Below is a concise table of state codes used to indicate officer status and immediate priorities. This page is a roleplay-themed, Police Scotland-styled example for public documentation.

| Code | Meaning | Description | Example |
| --- | --- | --- | --- |
| State 0 | Emergency / Officer in danger | Immediate life-threatening incident. Units should give highest priority and request urgent assistance. | "Unit 45, State 0, officer down at High Street" | 
| State 1 | Available | Officer is available for deployment. | "Unit 12, State 1, available" |
| State 2 | En route | Officer is travelling to an assignment. | "Unit 7, State 2, en route to incident" |
| State 3 | At scene | Officer has arrived at the location and is on scene. | "Unit 3, State 3, on scene at junction" |
| State 4 | Incident complete | Primary actions finished — returning to routine duties or awaiting tasking. | "Unit 21, State 4, incident clear" |
| State 5 | Meal break | Officer on authorised break. Not available for immediate emergency work. | "Unit 9, State 5, meal" |
| State 6 | Busy / unavailable | Engaged in an activity and not available for new tasks. | "Unit 2, State 6, dealing with custody" |
| State 7 | Prisoner on board | Transporting a detained person. | "Unit 11, State 7, prisoner on board, ETD 15" |
| State 8 | Returning to station | Officer heading back to station for admin or resupply. | "Unit 6, State 8, returning to base" |
| State 9 | Request urgent assistance | Immediate assistance requested — may be followed by location and nature. | "Unit 45, State 9, urgent assistance required" |

Each code above has a small copy button to make radio transmission and roleplay faster.

> Info: These codes are a simplified, roleplay-friendly adaptation and not an exact reproduction of any real police communication protocol.

## Examples

1. Officer reports an emergency:

> Unit 45: "State 0, officer injured, location: High Street"

2. Routine status update:

> Unit 12: "State 1, available"
